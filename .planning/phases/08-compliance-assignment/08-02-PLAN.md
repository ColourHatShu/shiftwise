## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Coordinator → Modal | Trusted action (OWNER/ADMIN role enforced at API layer) |
| Modal → API | Request body contains workerIds (validated on backend) |
| API → Frontend | Response contains compliance snapshot (immutable, read-only) |
| UI → Coordinator | Toast notifications inform of success/failure; no silent failures |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-07 | Tampering | Selected workerIds | Mitigate | Frontend sends workerIds, backend validates each exists and belongs to agency. Backend enforces role check. |
| T-08-08 | Spoofing | Coordinator identity | Mitigate | OWNER/ADMIN role check on POST /assign-bulk (Phase 1 auth). Frontend only shows button to authenticated users. |
| T-08-09 | Information Disclosure | Compliance snapshot displayed | Mitigate | Snapshot contains doc types (sanitized) and scores only, no PII or document numbers. Display restricted to OWNER/ADMIN. |
| T-08-10 | Denial of Service | Bulk select 1000 workers | Mitigate | Frontend limits checkbox to visible page (25 workers). Backend enforces max 100 workers per request. Rate limit: 100/15min. |

All mitigations aligned with Phase 1 auth and R-SA-02 compliance filtering.
