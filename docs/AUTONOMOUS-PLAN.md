## P16 ‚Äî Bug fixes (self-review)
- [x] **Deactivated workers were still assignable to shifts (candidate lists)** ‚Äî `deactivate` sets `status='INACTIVE'` (never touches `isActive`), but all three shift-assignment routes (`assignable-workers`, `assign-bulk`, `suggested-workers`) filtered only on `isActive: true` ‚Äî so a deactivated worker still appeared in the assignable list, bulk-assign candidates, **and** the shift-matcher "top picks". Added `status: 'ACTIVE'` to all three filters (matches the canonical filter used by reports/dashboard/deactivate) and a regression assertion on the suggested-workers where-clause. Found via a self-review bug-hunt of this session's code; fixes a real correctness/safety hole (could assign a worker the agency had deactivated).
- [x] **Deactivated workers were still assignable via the write path (defense-in-depth)** ‚Äî the candidate-list fix didn't cover the actual assign endpoints: `POST /assign` validated only agency membership (not status), and `POST /assign-bulk` used `validateComplianceForWorkers`, whose worker fetch had no status filter ‚Äî so a **direct API call** (or stale UI) could still assign a deactivated worker. Added an explicit `status !== 'ACTIVE'` ‚Üí 400 ("Cannot assign a deactivated worker") in `POST /assign`, and `status: 'ACTIVE'` to the validator's worker fetch (deactivated ‚Üí `notFound` ‚Üí skipped in bulk). +1 regression test; also fixed the happy-path test's worker mock (now needs `status: 'ACTIVE'`). `test:ci` 218.
- [x] **Off-by-one: documents expiring *today* were flagged overdue** ‚Äî `expiring-documents` computed `daysUntilExpiry = floor((expiryDate_midnightUTC ‚àí now_currentTime)/day)`, so a doc expiring today came out as `‚àí1` ‚Üí `overdue: true` / "Expired 1d ago" (and inflated the `overdue` count) even though it's still valid today. Fixed to a calendar-day diff (both normalized to UTC midnight, `Math.round`): today ‚Üí `0` / not overdue, yesterday ‚Üí `‚àí1` / overdue. +1 regression test (`test:ci` 219).
